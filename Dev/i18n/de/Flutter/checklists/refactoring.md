# Sichere Refactoring-Checkliste

## Vorbereitung

- [ ] Alle bestehenden Tests bestanden
- [ ] Baseline-Coverage aufgezeichnet
- [ ] Feature-Branch erstellt
- [ ] Backup/initialer Commit durchgeführt

## Während des Refactorings

- [ ] Eine Änderung nach der anderen
- [ ] Tests bestehen nach jedem Schritt
- [ ] Häufige Commits
- [ ] Keine funktionalen Änderungen

## Verifizierungen

- [ ] Alle Tests bestanden
- [ ] Coverage beibehalten oder verbessert
- [ ] Keine Performance-Regression
- [ ] flutter analyze sauber
- [ ] Kompilierung ohne Fehler

## Vor dem Merge

- [ ] Code-Review genehmigt
- [ ] Dokumentation aktualisiert
- [ ] CHANGELOG aktualisiert
- [ ] Integrationstests bestanden
